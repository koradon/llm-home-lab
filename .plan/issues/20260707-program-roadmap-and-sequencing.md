## Why

A top-level planning issue helps keep milestone sequencing explicit while syncing to GitHub.

## Scope

- Track delivery order: M1 -> M2 -> M3 -> M4.
- Record cross-milestone dependencies and blockers.
- Keep release readiness checklist for each milestone.

## Acceptance Criteria

- Dependency map exists for all milestone deliverables.
- Progress can be summarized from one issue during weekly reviews.
- Release checklist is updated before closing a milestone.
